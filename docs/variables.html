Variables & OUTPUT

The VARIABLE command is how you store and manipulate data in txts. Every variable belongs to a library namespace.

Appending a Value to a Variable

Use the VARIABLE command to append a value to a variable. The source value is added to the end of whatever the target variable currently holds:

IMPORT txts
VARIABLE txts.OUTPUT "Hello, "
VARIABLE txts.OUTPUT "world!"

This appends "world!" to txts.OUTPUT, which already holds "Hello, ". The result is "Hello, world!". Each VARIABLE call adds to, rather than replaces, the existing value.

Reading from Another Variable

You can append the value of one variable to another using a dotted reference as the source:

IMPORT txts
IMPORT lib
VARIABLE txts.OUTPUT lib.greeting

This reads the value of lib.greeting and appends it to txts.OUTPUT, adding it after anything already there.

Important: Both the target library and the source library must be imported before you can read from or write to them.
Append, not overwrite: VARIABLE always appends the source value to the target. If you call VARIABLE txts.OUTPUT "Hello " and then VARIABLE txts.OUTPUT "World", the result is "Hello World" — both parts are concatenated.

The OUTPUT Variable

txts.OUTPUT is the designated output channel. Whatever value is stored in it at the end of your program is printed to stdout (or written to a file if you specify one).

IMPORT txts
VARIABLE txts.OUTPUT "This gets printed"

Clearing a Variable with CLEAR

The built-in txts.CLEAR variable is a special sentinel. Unlike regular VARIABLE (which appends), txts.CLEAR overwrites the target variable to an empty string, letting you reset it:

IMPORT txts
IMPORT suite
VARIABLE suite.secret "This will disappear"
VARIABLE suite.secret txts.CLEAR
VARIABLE txts.OUTPUT suite.secret

After clearing, suite.secret is an empty string. The program would output nothing.

Variable Rules

  • Variable names are case-sensitive
  • Variables are always namespaced to a library (use dot notation)
  • VARIABLE always appends — the source value is added to the end of the target's current value
  • The only exception is txts.CLEAR, which overwrites the target with an empty string
  • You can only assign to variables in libraries you have imported
  • The txts library is read-only except for txts.OUTPUT
  • Variables persist for the lifetime of the program execution

Creating Variables in Your Own Library

To create variables for temporary storage, use a library namespace that you control. Convention is to IMPORT your own library name:

# In myproject/main.txts
 IMPORT myproject
IMPORT txts
VARIABLE myproject.temp "temporary value"
VARIABLE txts.OUTPUT myproject.temp

Value Sources

The source value in a VARIABLE command (the value being appended) can be:

TypeExampleDescription
String literal "hello" A double-quoted string value
Variable reference lib.greeting Read from another variable in an imported library
