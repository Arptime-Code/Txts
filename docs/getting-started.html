Getting Started with txts

Welcome to txts! This guide will help you set up the runtime and write your very first program.

Prerequisites

txts runs on Node.js. Make sure you have it installed:

node --version

You need Node.js v12 or later.

Setting Up a Project

Create a new directory for your project and initialize it:

mkdir my-txts-project
cd my-txts-project
npm init -y
Note: Txts finds your project root by looking for a .local-txtspm/ directory. No configuration files are required.

Your First Program

Create a file named hello.txts:

IMPORT txts
VARIABLE txts.OUTPUT "Hello, world!"

Run it with the txts interpreter (from inside the Txts repo):

node bin/txts hello.txts

Or if the repo is somewhere else on your system:

node /path/to/txts/bin/txts hello.txts

You should see:

Hello, world!

How It Works

Let's break down the two lines:

  • IMPORT txts — Imports the built-in txts library, which gives you access to special variables like txts.OUTPUT.
  • VARIABLE txts.OUTPUT "Hello, world!" — Appends the string "Hello, world!" to txts.OUTPUT. Whatever value is in txts.OUTPUT when the program finishes is printed to the console. Multiple VARIABLE calls accumulate — each one appends rather than overwriting.
Important: Every file must IMPORT any library it uses, including txts. Imports are per-file.

Output to a File

You can also redirect output to a file instead of stdout:

node /path/to/txts/bin/txts hello.txts output.txt

This writes the output to output.txt instead of printing it.

CLI Flags

The interpreter also supports these flags:

node bin/txts --help # Show full usage information
 node bin/txts --version # Print version number
